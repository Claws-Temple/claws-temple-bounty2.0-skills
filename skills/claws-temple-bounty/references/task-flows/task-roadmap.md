# Task Roadmap Flow

Use this flow for generic bounty questions.

## Goal

Explain the full path and route the user to the next task without overloading them.

## Steps

1. State the five-task path in branded language.
2. Mark Task 1 through Task 4 as the main qualification path.
3. Mark Task 5 as optional community reach.
4. If no progress is known, recommend Task 1.
5. If some progress is known, recommend the next unfinished task.

## Required Visible Output

- current task roadmap
- one-line purpose for each task
- qualification note: Task 1 through Task 4 unlock the Claws Temple Bounty 2.0 qualification path
- optional note: Task 5 is extra reach
- direct CTA into the recommended next task

## Maintainer Notes

- keep the visible layer short
- do not expose dependency skill names in the visible layer
- if the user asks for details, you may name the internal dependency flow in the maintainer layer
